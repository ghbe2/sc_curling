### 1.2 完成している物理・操作（すべて `proto23_takeout.html` 内）

- ストーン選択とチュートリアルの上端を揃える。長押し早送り「はやく♡」は投球後の発射位置に表示する。
- 初心者向け投球チュートリアルは操作段階に連動する。最初は指を下へ引く動きだけを示し、狙いモードへ入ったら左右の動きと「決まったら離す」に切り替える。投球は上へ押す動きと速度（はやい・ふつう・おそい）を示す。回転ゲージは中央から左右へ伸ばし、「左に曲がる・右に曲がる」と結果を表示する。
- 投球開始後は照準欄を「後ろに下げてやめる」に切り替える。チュートリアルは最小化でき、その設定を端末に保存する。投球軌道は扇形ではなく、回転方向へ曲がる一本の弧で表示する。
- 軌道ガイドは固定形状ではなく、現在の速度・回転・キャラ補正・摩擦・転がり抵抗を本番と同じ式で事前計算する（衝突後の軌道は対象外）。
- ハウス中央のボタンに上下へふわふわ動く💖を置き、「💖を守れ！」を常時表示してゲーム目的を明示する。
- 物理予測線は速度によって長さ、回転によって曲がり方を変える。背景や弱パワーでも消えない濃い紫＋白縁の実線とする。チュートリアル開閉ボタンは枠外下部の小さな「投げ方」とする。
- 投球後のデバッグ軌跡は廃止する。実軌道ガイドは照準決定直後から投球リリースまで表示し、入力中の押し量・横ずれに追従する。
- パワーは照準時の引き量から切り離し、投球時に前へ押すほど速くする。最初の58px（約1〜2石分）は最弱を維持し、そこから145pxまでの範囲で最大へ上げる。
- 敵ストーンは黒背景で描画する。敵接近時は味方への押し出しを2倍にし、衝突後も敵が接触前の進行速度を約80%残して押し込む。ガードによる遅延は残すが、💖上の味方1個で鉄壁にはならない強さとする。
- 目的表示は「敵を弾いて💖を守れ！」とする。チュートリアル展開時のボタンは「かくす」、最小化時は「投げ方」とする。
- 早送りボタンの表示はアイコンなしの「はやく♡」とする。ガイドは停止位置を明かさず画面奥まで一定長で描き、速さは青（遅い）から赤（速い）の線色だけで表現する。
- 回転は弱いほど大きく曲がり、強いほど直進する。実物理とガイドは同じ逆比例変換を使う。チュートリアルは「左回転・右回転」のみ表示し、中央時の「まっすぐ」は表示しない。
- 方向ガイドは左右対称の2本線とし、その間を現在の速度色で薄く塗る。入力した回転方向側を濃く、反対側を薄くする。遅いほど範囲が大きく曲がり、速いほど直進寄りにし、強回転ほど左右の範囲を狭める。
- ガイドの左右線は、初速の65%まで減速する滑走区間を投球本体と同じ物理で進める。その後は停止時の方向ではなく、滑走区間末尾の曲率を保った円弧として画面奥へ延長し、停止位置は明かさない。
- 上記のカーブ軌道予測ガイドは調整保留として一時停止する。現行表示はカーブ・速度・停止位置を含まない、照準方向だけの一本の直線ガイドとする。ユゥナリナの軌道予測も一時停止する。
- ハウス侵入の第一警戒表示は英語を使わず「ちかづいてきた！」とする。
- 「どれみふぁそらしど」の衝突音と8音ボーナスは停止する。衝突音は今後の演出刷新で差し替える。
- **投球操作**（スリングショット式）: 引く → アンカー（石の元位置）を越えて前へ押し出す → 離す。
  - 引き距離 = 基礎パワー、押し出し距離 = パワー微調整（短い=強い / 長い=弱い、前後スワイプで双方向に調整可）。
  - 横ズレ = 回転量（カーブ）。狙いの中心線からの符号付き垂直距離で決まる。
- **パワー**: 最低速度は端末の画面比率に依存しないよう**ランチャー距離から動的に逆算**（後述）。
- **カーブ**: 回転量に比例。速いほど直進、減速する終盤ほど強く曲がる（実カーリングのドロー感）。
- **衝突**: ビリヤードの「90度ルール」を土台にした**入射角ベースの力・回転の移行**（後述）。
- **早送り**: 「盤上で動いている石が全てほぼ停止」したら、止まりかけの無駄時間だけを最大9倍で圧縮（物理結果は不変）。
- **表示**: カーリングのハウス（青/白/赤/ボタン）+ デバッグ用グリッド（40px間隔）+ 軌跡トレイル + 衝突演出（画面揺れ・瞬間ズーム）。

### 1.3 検証方法（このプロジェクト特有）
- サンドボックスのブラウザは `requestAnimationFrame` の実時間プレイが不安定。**ローカルHTTPサーバ + ブラウザのJSコンソールから `physicsStep()` を直接ステップ実行**して着地点などを数値検証するのが主手段。
- 手描きの軌道イメージや数値チャートを「校正点」として、シミュレーションで数値を合わせ込むスタイル。

### 1.4 主要 CONFIG（PROTO23 現在値・チューニングの核）
物理の「手触り」はここに集約。**キャラ補正はこの値を直接書き換えず、個体ごとの倍率/差分で上書きする**方針にする（1.5 参照）。

| 定数 | 値 | 意味 |
|---|---|---|
| `BOARD_RADIUS` | 320 | 盤（シート）半径 |
| `HOUSE_RADIUS` | 130 | ハウス（的）外周半径 |
| `GRID_SPACING` | 40 | デバッグ用グリッド間隔 |
| `PIECE_RADIUS` | 24 | 石の半径（当たり判定の基準） |
| `FRICTION` | 0.985 | 1フレームあたり速度減衰 |
| `STOP_DIST_PER_SPEED` | 65.35 | 実測: 直進の石の停止走行距離 ÷ 初速。停止距離 ≒ 初速×65.35 |
| `MIN_THROW_TARGET_R` | 210 | 最弱・直進の目標停止半径。ここから動的に最低速度を逆算 |
| `MIN_THROW_SPEED_FLOOR` | 6 | 動的最低速度のさらに下限（安全弁） |
| `MAX_THROW_SPEED` | 46 | 初速上限 |
| `DRAW_TO_SPEED_SCALE` | 0.5 | 引き距離→基礎初速 |
| `PUSH_MAX_POWER_RANGE` / `PUSH_SOFT_RANGE` / `PUSH_MIN_COEF` | 15 / 100 / 0.08 | 押し出し→パワー係数曲線 |
| `CURVE_STRENGTH` | 0.011 | 回頭の基準強度（curveに単純比例） |
| `CURVE_VREF` / `CURVE_SLOW_POWER` | 8 / 2.4 | 速度依存（減速する終盤ほど強く曲がる） |
| `CURVE_MAX_ANGLE` | 1.4 (rad) | 1投あたりの累積回頭角の上限（≒80°） |
| `CURVE_MIN` | 0.1154 | 最小回転（無回転の排除。ほぼ直進の微カーブ） |
| `SPIN_VISUAL_MIN_RPS` / `MAX_RPS` | 0.3 / 1.2 | 見た目の自転速度（実時間rps基準、滑走時間非依存） |
| `INCIDENCE_TRANSFER_POWER` | 2.0 | 並進衝突: 入射角の底上げ指数（中間角度で移行を増やす） |
| `SPIN_TRANSFER_MIN_IMPACT` | 2 | 回転伝達が起きる最小衝突速度 |
| `TIME_SCALE` | 0.2625 | 画面上の滑走再生速度（物理は不変、重量感の演出） |
| `FF_SLOW_SPEED` / `FF_MAX_MULT` / `FF_EASE` | 0.5 / 9 / 0.06 | 早送りのしきい速度 / 最大倍率 / 補間率 |

**動的最低速度**: `minThrowSpeed()` = `(D - MIN_THROW_TARGET_R) / STOP_DIST_PER_SPEED` を `[MIN_THROW_SPEED_FLOOR, MAX_THROW_SPEED]` でクランプ。`D` = ランチャーから盤中心までの距離。

**入射角ベースの衝突（要点）**: 速い側=S（投げた石）、遅い側=T（当てられた石）。S の相対速度と接触法線のなす角θで配分。θ=0（正面）で S 停止・T が全速直進、θ=90°（かすり）で双方ほぼ無反応。T は入射方向へ `relSpeed*cosθ'`、S は接線方向へ `relSpeed*sinθ'`（θ' は `INCIDENCE_TRANSFER_POWER` で底上げした実効角）。回転は入射角チャート（`SPIN_ANGLE_DEG=[0,30,45,75,90]`, 自残 `[.9,.6,.3,.05,0]`, 相手移行 `[0,.2,.5,.4,0]`）を区分線形で適用。

### 1.5 キャラ補正を差し込む「フック地点」（`proto23_takeout.html` の関数/行）
実装者向け。各石に個体パラメータ（例 `p.mods = {...}`）を持たせ、以下の箇所で参照する。

| 補正したい挙動 | フック地点（関数・目安行） | 現在の式 |
|---|---|---|
| 投球速度（速い/遅い） | `finalizeAim()` L571 | `speed = clamp(base * pushCoef(pushDist), minThrowSpeed(), MAX)` → `* p.mods.speedMult` |
| カーブの効き（曲がりやすさ） | `physicsStep()` 回頭 L636 | 回転入力 `p.curve` は全員共通。軌道だけ `* p.mods.pathCurveMult` で補正 |
| 力の移行（当てられ側の飛びやすさ） | 衝突 L722-723 | `sSpeed`/`tSpeed` に個体係数。T の受け取り減=`tSpeed *= T.mods.knockResist`（ベルペット等） |
| 自分の止まりやすさ | 衝突後の S 速度 L722 付近 | `sSpeed *= S.mods.selfStopMult`（ラコ等、衝突後に自分を追加減速） |
| 反発（ぼよよん） | 衝突 L722-723 の後 | 反発成分を法線方向に加算（瑞珠。現モデルは反発ゼロなので新規に足す） |
| 部位当たり判定（耳/角） | 衝突判定 L680 `minDist` | 石の向き `p.rot` と部位方向から、方向依存で当たり半径を拡張（新規：形状判定） |
| エネミーの吸い込み（ヒロイベオ） | エネミー移動ロジック（新規） | 敵の移動ベクトルにヒロイベオへの引力を加算 |

> 注意: 現在の並進衝突モデルは**等質量前提**。「重さ」を本格導入するなら質量比を衝突式に入れる拡張が必要。まずは上記の係数補正（速度の移行量を個体倍率でいじる）で"重さ風"を表現するのが低コスト。

## 2. 次フェーズ仕様：サバイバルモード

### 2.1 コアループ
- **1ショット = 1ターン**。
- プレイヤーは毎ターン**手札4個**のキャラストーンを持ち、そこから1個選んで投げる。投げるたびに**ランダムで1個補充**（常に4個）。
- 毎ターン、周囲からエネミーストーン（七つの大罪）が**少しずつ中心に近づいてくる**。

### 2.2 エネミーストーン（七つの大罪）
- 移動方法は**リアルなカーリングと完全に別物でよい**（毎ターン中心へ一定量ワープ/スライドで近づく等、抽象でOK）。
- ただし、プレイヤーが中心に置いたガードを崩すため、**エネミーもある程度こちらの石を弾いてくる必要**がある（強さ・頻度は遊びながら調整）。
- エネミー自身の個性付けは**後回し**。まずは「近づく・時々弾く」だけ。

### 2.3 勝敗条件
- **中心（ボタン/ハウス中心）に七つの大罪が「止まる」と負け**。
- ただし**1ターンの通過はOK**（止まって居座ったら負け）。＝「真ん中の敵石を弾き出す」テイクアウトがカーリング体験の核。
- 自分の石を中心に置くと強力なガードになる（＝上記2.2でエネミーがこちらを弾く動機になる）。

### 2.4 スコア
- 生存ターン数
- 弾き出したエネミー数
- 盤上の自分の石の数 × 中心からの距離（中心に近いほど高得点）
- これらを合算して伸びていく形。
- 盤上の敵を全滅させると2000点のWAVE CLEARボーナスを加算し、七つの大罪7人を外周へ再配置する。
- 各ウェーブで敵が12フット・8フット・4フットの境界へ初めて到達したとき、外周へ敵を1人ずつ増援する。
- 増援判定は敵ごとに行い、同時に複数の敵が境界へ侵入した場合は同数を追加する。増援時の告知は出さない。
- 敵が7人未満になるほど1ターンの接近距離を伸ばし、1人時は基準の1.8倍とする。
- 旧盤面半径320を越えても石を残し、半径520を越えた場合のみテイクアウトとする。盤面表示はハウスだけとする。
- 同一投球で2体・3体をテイクした瞬間に、中央へ「だぶるていくっ!!」「とりぷるていくっ!!!」を表示する。4体以上でも専用の「よんこどり」は出さない。
- 投げた石が別の石を動かし、その石が敵を場外へ飛ばした場合は、敵が場外へ出た瞬間に「れいずていくっ！」を表示する。
- 投げた石を他の石へ当てずボタンへ直接止めた場合は、停止時に「ぼたんどろーっ！」を表示する。
- 特殊ショット表示に加点内訳は載せない。全滅時は「ぶらんくっ！」を表示して全滅ボーナスを加算する。
- 新しいプレイヤーターン開始時の盤面だけで警戒判定を行う。敵がハウスへ掛かっていれば第1警戒、ボタンへ掛かっていれば `PINCH!` の第2警戒。移動中の一時通過では警告を出さない。ボタン到達直後は敗北にせず、次のプレイヤーショット終了時にもボタンから退けられていなければGAME OVERとする。
- ストーンの物理判定は円形のまま維持し、プレイヤーと七つの大罪の見た目は各キャラクターの絵文字を表示する。

### 2.5 キャラストーン（十二支メイン）の特徴 → 物理補正マッピング

### 2.5.1 七つの大罪の固有能力
- **ウーヨン**: 通常の1.55倍移動し、移動後に半径92px内の周囲の石を外向きへ弾く。
- **ウェンカ**: 敵移動中に味方石へ接触すると、そのターン1回だけ隣へ小型分身「ムイ」を生む。
- **デロルス**: 接触した別の敵を1.35倍へ大型化し、被弾速度を半減させる。
- **アルバ**: 投げた石による直接テイクを防ぐ。別の石を経由したレイズテイクだけ有効。
- **エノ**: 接触した味方石を0.68倍へ小型化し、他の石を弾く力を0.45倍へ弱める。
- **マルガリティフェラ**: レイズテイクを防ぐ。投げた石を直接当てたテイクだけ有効。
- **ジーナ／マミ**: ターンごとに姿が交互に変わる。敵移動後、ジーナは周囲の敵、マミは周囲の味方石を自分へ引き寄せる。

衝突された石の初速方向は、打った石の進行方向ではなく、**接触点と衝突された石の中心を結ぶ法線方向**とする。その後、受け渡された回転エネルギーによって通常のカーブがかかる。ユゥナリナの命中後予測線も同じ法線方向を表示する。

#### 軌道予測（ユゥナリナへ実装済み）
- 狙っている間、最初に当たる石と、衝突後にその石が飛ぶ方向をガイド表示する。
- 現段階は最初の1衝突のみ。複数衝突やカーブ込みの精密予測は必要になった時点で相談する。

### 2.6 実装の進め方（段階）
Codex での推奨順序。各段階で `physicsStep()` 等の既存物理には手を入れず、上に載せる形で。

1. **ターン管理 + 手札システム**: 4個ランダム所持、選択して投げる、投げたら補充。まだキャラ補正なし（全部マユゥ=基準）でOK。
2. **エネミー出現 + 中心接近 + 勝敗判定**: 七つの大罪を周囲に配置、毎ターン中心へ寄せる、中心居座りで負け。エネミーの「弾き」は最初なしでも可。
3. **スコア表示**: 生存ターン・テイク数・盤上石の中心距離。
4. **キャラ補正レイヤー（簡単な4個から）**: 回転入力と軌道補正を分離し、ミナリ・アウネは `pathCurveMult` で曲がり方だけを変更。ユゥナリナは0.8倍サイズ+初回衝突軌道ガイド、モコクンは1.2倍サイズ。
5. 衝突係数系（ベルペット・ラコ・瑞珠）。
6. 部位当たり判定系（バーニ・リウ・ぱふ）＝形状判定の新規実装が要るので最後。
7. ヒロイベオ（エネミーAIに引力）。
8. エネミー自身の個性付け。

> **参謀メモ:** 段階4までで「サバイバルとして遊べる最小形」になる。部位判定（バーニ/リウ/ぱふ）は `p.rot`（石の向き）を当たり判定に使う新規実装で、既存は `rot` を見た目にしか使っていないため一段重い。まず段階1〜4を回して手触りを確認してから着手するのが安全。
